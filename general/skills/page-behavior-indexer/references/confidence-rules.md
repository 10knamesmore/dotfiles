# 置信度规则

为每个交互标注置信度。

## High

- 触发、参数、结果在代码中都明确可见。
- 有明确 handler 调用 mutation/导航，并伴随清晰 UI 反馈。
- UI 结果可观察且与动作直接关联。

## Medium

- 触发明确，但参数或结果部分推断。
- 有 handler，但结果依赖命名或间接效果。
- 可能存在多个结果，其中一个更可能但不显式。

## Low

- 仅凭文案、组件名或视觉暗示推断。
- 参数未知或猜测。
- 结果高度不确定或仅基于惯例。

## 降级规则

- 关键字段（触发/参数/结果）只要有推断，至少降一级。
- 自动触发但范围不清晰（如 on mount + 全局副作用），优先 medium/low。
- 同一 UI 元素对应多个 handler 且意图不清，优先 low。
