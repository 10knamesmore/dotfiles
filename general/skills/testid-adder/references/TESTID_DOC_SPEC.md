# 完备 testid 文档规范（Agent 版）

目标：让 Agent 在尽可能不阅读源码的情况下，仅凭文档即可定位元素并写出 Playwright PageObject。

## 必含结构
0. 自然语言描述的页面功能概览  
   - 这个页面可能的自然语言描述, 比如 T0-Trade, 可能标题为"底仓增强交易页面", "T0交易页面"
   - 简要说明页面核心功能与交互流程。
   - 说明组件根元素文件(tsx)路径(以src/开头), 以便 Agent 定位源码。
1. **页面组件树（组合化）**  
   - 以层级关系列出页面 → 区域 → 组件 → 子组件。
2. **testid 前缀与规则**  
   - 说明 page 前缀、组件前缀、table cell 前缀、dialog 前缀等。
3. **组件级映射清单（自然语言 → testid）**  
   - 每个可交互元素与可断言数据必须列出。
   - **注意** 所有有testid的元素都必须列出, 就算这个组件不是这次改动的范围.
   - 如果是交互方式为显示文本的text类, 需要列出示例文本, 比如`0.05%`, `1,000,000`。
   - 区域的组件包含组件 根元素文件 路径(以src/开头)。
4. **覆盖缺失说明**  
   - 明确缺失点与兜底定位策略（如 placeholder / 文本定位）。
5. **Playwright PageObject 示例**  
   - 页面根类 + 关键子组件类 + 2-3 个常见操作方法。
6. **样例用例片段**  
   - 证明文档可直接指导 Agent 生成测试。

## 同步要求
- 所有文档放在项目根目录的 mapping 目录(这个目录可能是一个软链接, 你允许对其进行读操作, 但写操作前需要询问用户)下
- 每次新增/修改 testid **必须同步更新对应文档**。
- 文档更新与代码变更 **必须在同一次提交内完成**。

## 表格组件强制规则
- 所有表格组件的**每个 cell 必须有 testid**，包括：
  - 自定义 cell 渲染
  - 纯文本 cell
