# TypeScript 文档注释规范（TSDoc/JSDoc 风格）

## 基本形式

- 使用 `/** ... */` 作为文档注释。
- 适用于导出的函数、类、接口、类型别名、枚举、常量、React 组件与公共方法。
- 注释使用 Markdown 语法，支持代码块。

## 放置位置

- 文档注释紧贴在被注释声明上方。
- 对于类方法：仅在公共或关键业务方法上添加；私有方法可省略。

## 必要字段

- 简短的首句描述用途（动词开头更清晰）。
- 当参数或返回值语义不直观时，使用标签补充。

## 常用标签

- `@param`：说明参数语义、单位、范围、默认值或可选条件。
- `@returns`：说明返回值含义。
- `@throws`：说明可能抛出的错误。
- `@example`：给出最小使用示例。
- `@deprecated`：标记弃用原因与替代方案。
- `@remarks`：补充行为细节或边界条件。

## React 组件（.tsx）建议

- 说明组件用途与主要交互。
- 对 `Props` 中关键字段使用 `@param` 说明；可在 `Props` 接口上说明整体语义。
- 提供简短示例，避免冗长模板。

## 风格与边界

- 避免与函数/类型名重复的空洞描述。
- 保持同一文件内风格一致（是否使用标签、示例格式、换行风格）。
- 不要引入与行为无关的实现细节。
